# Contribution Guidelines

Hello! Thanks for your interest in improving the EraVM Airbender verifier. 

All contributions are subject to review, and not every contribution is guaranteed to be merged.
For significant changes, please open an issue first so maintainers can confirm that the proposed
work fits the project direction and maintenance capacity.

> [!WARNING]  
> Due to reduced capacity, Pull Requests that are not associated with an issue
> with an explicit approval from maintainers are unlikely to be reviewed/merged.
>
> We do not currently accept non-functional/stylistic contributions.

## Ways to contribute

1. Open issues for bugs, missing functionality, or improvements.
2. Add context to existing issues with logs, reproduction steps, command output, or relevant batch numbers.
3. Resolve issues by demonstrating that the current behavior is correct or by opening a focused pull request.
4. Report security issues through the process in [our security policy](./SECURITY.md).
5. [Join the team](https://matterlabs.notion.site/Shape-the-future-of-Ethereum-at-Matter-Labs-dfb3b5a037044bb3a8006af2eb0575e0).

## Fixing issues

To contribute code, fork the repository, make a focused change, add or update tests and documentation when relevant, and open a pull request. GitHub has [guidance for pull requests from forks](https://docs.github.com/en/pull-requests/collaborating-with-pull-requests/proposing-changes-to-your-work-with-pull-requests/creating-a-pull-request-from-a-fork).
